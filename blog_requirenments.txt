Требования к блогу библиотеки

Первостепенные требования:

1. Блог должен быть разработан в виде веб-сайта и включать такие модули:
- регистрация новых пользователей;
- авторизация пользователей;
- главная страница блога (со статической общей информацией о библиотеке, ее блоге и правилам общения пользователей);
- раздел постов с новостями библиотеки;
- раздел постов о событиях библиотеки;
- раздел постов о ресурсах библиотеки;
- личный кабинет для авторизированного пользователя, в котором будут отображаться:
     + посты данного пользователя;
     + комментарии, оставленные данным пользователем.

2. Блог должен позволять пользователю:
- зарегистрироваться с помощью адреса электронной почты, указав:
     + ник ([3-15] символов без пробелов);
     + желаемый пароль ([6-25] символов без пробелов);
     + подтверждение пароля;
- авторизоваться с помощью:
     + адреса электронной почты ИЛИ ника;
     + и своего пароля.

3. В каждом из 3-ех разделов блога (см. пункт 1 о разделах) пользователи имеют возможность:
- создавать и публиковать свои посты;
- удалять свои посты.

4. Блог должен позволять пользователю создавать пост, включая такие параметры:
- ник пользователя (добавляется системой автоматически);
- дата и время публикация поста (добавляется системой автоматически);
- раздел поста (добавляется системой автоматически в зависимости от того, в каком из разделов пользователь решил создать пост);
- тема поста (добавляется пользователем, [2-30] символов);
- текст поста (добавляется пользователем, [5-1000] символов).

5. Блог должен позволять пользователю удалять пост, при этом удаляя все оставленные комментарии к данному посту.

6. Блог должен позволять пользователю:
- комментировать свои посты;
- комментировать посты других пользователей;
- удалять свои комментарии.

7. Блог должен позволять пользователю создавать комментарий, включая такие параметры:
- ник пользователя (добавляется системой автоматически);
- дата и время добавления комментария (добавляется системой автоматически);
- текст комментария (добавляется пользователем, [1-400] символов).

8. Поддержка веб-сайта должна быть реализована для таких веб-браузеров:
- Google Chrome версий 60, 59, 58, 57;
- Internet Explorer версий 11, 10, 9, 8;
- Microsoft Edge версий 41, 40;
- Mozilla Firefox версий 55, 54, 52.

9. Стабильность системы должна обеспечиваться следующим образом:
- минимальное количество пользователей, которые одновременно пользуются веб-сайтом – 100 пользователей;
- время перехода на каждую страницу веб-сайта не должно превышать 10 секунд;
- система должна поддерживаться в рабочем состоянии 24 часа 7 дней в неделю.

Второстепенные требования:

10. Должна быть разработана мобильная версия блога в виде оптимизации веб-сайта для мобильных устройств:
- для браузеров Safari (for iOS) и Google Chrome (for Android);
- и для наиболее популярных размеров экранов смартфонов: 
     + iPhone 6/6S/7 - 375x667
     + Samsung Galaxy S3/S4/S5/S6/S7 – 360x640
